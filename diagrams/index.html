System Architecture

Swarm Conductor 分为五个层次：Conductor 调度核心、Agent 运行时、持久化层、工作区隔离、CodeWhale Agent Pool。Conductor 是外部编排层，不修改 CodeWhale 本身，每个 CodeWhale 实例通过 HTTP API 接受调度。

Conductor Layer

调度核心：TaskDAG、FileLockRegistry、ApprovalGate、CrashRecovery

Agent Runtime

进程池管理 + CodeWhale HTTP/SSE 客户端

Persistence

SQLite WAL 模式，存储 runs/tasks/memory/event_log

Workspace

文件锁 + Git 分支隔离，防止并发冲突

CodeWhale Pool

最多 20 个独立进程，每个监听不同端口

Core Execution Flow

从用户输入到任务完成的完整链路。关键点：markBusy 在第一个 await 之前同步执行（防双重调度）；SSE 事件流只收集 kind=agent_message；任务完成后立即写共享记忆供下游 agent 读取。

调度循环

tick() 每 500ms 执行一次，检测冲突和死锁，把 ready 任务分配给 idle agent

并行执行

dispatch() 是异步的，多个任务可以同时进行，互不阻塞调度循环

SSE 事件流

通过 Server-Sent Events 实时接收 agent 输出，turn.completed 作为终止信号

输出解析

解析 ## SUMMARY / CHANGES / EVIDENCE / RISKS / BLOCKERS 五个 section

Task DAG & State Machine

左侧：Task 的七种状态及转换规则。右侧：swarm run --goal 自动生成的五阶段 DAG 示例，展示扇入（多个 explore 汇聚到 plan）、扇出（plan 触发并行的 implement+review）、以及动态任务插入机制。

状态机关键点

blocked -> ready：最后一个阻塞依赖完成时触发；failed 会重试最多 2 次

依赖解析

addTasks() 批量添加时自动计算初始状态并连接双向边（dependsOn / blocks）

并行策略

无依赖的任务立即变为 ready，调度器在同一 tick 内可分配多个任务

动态扩展

BLOCKERS 自动插入 implement 任务，RISKS (HIGH) 插入 review 任务，去重保护
